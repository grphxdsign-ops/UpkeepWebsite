# Upkeep Site Map And Design Purpose

## Core Brand Idea

Upkeep is a virtual bookkeeping SaaS for people who want clean accounting records without learning complex accounting software.

The site should mesmerize visitors with one simple transformation:

**Document enters -> numbers become clear -> records and forms are ready.**

Do not make OCR the headline idea. The visitor does not need to understand the technology first. They need to feel, quickly, that the paperwork burden is handled.

## Primary Visitor Purpose

1. Arrive with paperwork anxiety or curiosity.
2. See that Upkeep turns messy documents into usable records.
3. Understand the process in seconds.
4. Trust that the system is simple enough for non-accountants.
5. Request access or book a demo.

## Page Map

### 1. Home Hero

Purpose: Capture attention immediately.

Visual: A document is scanned or pulled through a beautiful transformation object. Numbers lift from the document and settle into a clean form or record.

Copy role: Short promise only. Avoid dense explanation.

### 2. Process

Purpose: Show the simple flow.

Steps:
- Scan or upload documents.
- Upkeep reads and organizes the useful numbers.
- Forms and records become ready for review, storage, or export.

Animation target:
- 2D state-machine motion in Rive.
- Smooth scroll narrative with Lenis.
- If needed, a Spline object for a physical scanner or ledger artifact.

### 3. Outputs

Purpose: Make the result tangible.

Show:
- Receipts.
- Statements.
- Work orders.
- Simple forms.
- Searchable records.

Design rule: Do not turn this into a dashboard wall. Use one strong output visual with a few concrete examples.

### 4. Workspace Preview

Purpose: Build trust that the tool has a real working surface.

Show:
- Review queue.
- Flagged details.
- Ready records.
- Export path.

Design rule: Minimal, calm, functional. This is where product UI can appear, but only after the visitor understands the transformation.

### 5. Use Cases

Purpose: Let visitors self-identify.

Audience examples:
- Solo business owners.
- Small business operators.
- Property managers.
- Service businesses.
- Bookkeepers who want faster intake.

### 6. Trust And Simplicity

Purpose: Remove fear.

Messages:
- No complex software.
- No accounting knowledge needed.
- Documents become usable records quickly.
- The user stays in control through review.

### 7. Final CTA

Purpose: Convert interest.

CTA options:
- Request access.
- Book a demo.
- Join waitlist.

Default until confirmed: Request access.

## Animation Stack

- **Spline:** 3D only when a true physical hero object is necessary.
- **Rive:** 2D process and state-machine animations.
- **Shader Gradient:** Animated gradient fields and atmosphere.
- **Lottie Files:** Small animated icons and micro-confirmations.
- **Icons8:** Animated illustration references when a human-friendly explanatory asset is needed.
- **Lenis:** Smooth scroll.
- **Barba.js:** Page transitions once multiple pages exist.

## Design Reference Roles

- **Dark.design:** Use only for sleek moody lanes.
- **Godly.website:** Use for modern, high-impact web composition.
- **Minimal.gallery:** Use for the quiet minimalist lane.
- **Webflow:** Use as an implementation constraint for exportable sections and interaction patterns.
- **Neuronsinc:** Use for attention checks: one focal point, low clutter, clear CTA path.
